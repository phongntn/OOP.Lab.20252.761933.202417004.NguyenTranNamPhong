Question:
When should accessor methods be used?
Answer:
The author's main argument is that overuse of getter and setter methods exposes information of the object, turning it into data containers and weakening encapsulation rule.
So, only use accessor methods when necessary for exposing information.
Question:
If you create a constructor method to build a DVD by title then create a constructor method to build
a DVD by category. Does JAVA allow you to do this?
Answer:
No, we can't since JAVA recognise constructors by number of parameters, types of parameters and order of parameter types, so `title` and `category` will be recognized as DVD(String).
